**Key Technical Concepts**
-   **ID Gap-filling and Reuse:** An algorithm that queries existing assets, parses their numeric IDs, finds the first missing integer in the sequence, and serves it as the next available ID.
-   **Audit Trail/History Logging:** Storing a chronological array of events (created, deleted, reassigned) for each unique ID in a dedicated MongoDB collection ().
-   **Real-time Frontend Validation:** Using  or  events in React to trigger an asynchronous API call to a validation endpoint, providing immediate UI feedback (e.g., error messages, disabled buttons) to the user before form submission.
-   **Client-Server Separation of Concerns:** Shifting the responsibility of ID generation entirely to the backend to ensure data integrity, with the frontend only displaying placeholders until the data is committed.
