<product_requirements>
The primary goal is to transform the admin portal into a fully multi-tenant application. A super-admin user must be able to view data in two modes:
1.  **All Customers View:** When All Customers is selected in a global tenant switcher, all data across different parts of the application (Locations, Inventory, Orders, etc.) should be displayed in an aggregated view. For instance, the locations table should list locations from all tenants and include a column specifying which tenant each location belongs to.
2.  **Single Tenant View:** When a specific tenant (e.g., Europcar) is selected, the entire admin portal interface must filter to show *only* the data associated with that tenant. Selecting a tenant in the switcher should also automatically navigate the user to that tenant's main dashboard page.

Additionally, the UI for the location detail page should be enhanced with several key performance indicator (KPI) tiles to display statistics like total scans, user counts, device status, and open support tickets for that specific location.
</product_requirements>
